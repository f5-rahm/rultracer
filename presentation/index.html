Bytecode reference — opcode meanings & why some commands don’t appear

Why don’t I see some common Tcl commands (e.g. set, append) in the trace? The Tcl interpreter compiles the script to byte code before the first execution, and some Tcl commands are replaced by the byte codes — so they never appear as CMD occurrences. Their effects still surface: a variable write shows up as VAR_MOD, and a command invocation as CMD_VM/CMD. Bytecode occurrences also carry only the instruction mnemonic — rule-profiler does not emit the operand/value being pushed.

opcodemeaning
push1push a literal/constant onto the stack
loadScalarStkload a scalar variable’s value
storeScalarStkstore into a scalar variable (surfaces as VAR_MOD)
invokeStk1invoke a command (the following CMD_VM names it)
concat1concatenate strings on the stack
eqequality comparison (==)
doneend of the script’s byte code

Further reading: iRule Execution Tracing & Performance Profiling, Part 3 · An Abstract View of iRules with the Tcl Bytecode Disassembler — use Tcl’s disassembler to recover the literals/operands the trace omits.
